# Assets

本目录存放项目级静态资源和参考图片，主要用于文档、方案说明、传感器点位说明和非前端运行时的资源归档。

## 目录用途

- `assets/images/`：从历史根目录整理过来的登录、注册等参考图片。
- `assets/equipment-images/`：传感器、设备、化工园区场景等说明图片。

前端运行时必须直接加载的资源应优先放在 `frontend/public/` 或 `frontend/src/assets/`，不要在页面中跨目录引用本目录文件。

## 提交规则

- 必须能说明资源来源、用途和授权状态。
- 临时截图、草稿图、聊天生成过程图、重复图片和无用途图片不得提交。
- 大体积视频、模型文件、未压缩图片应优先放到外部对象存储或发布包，不直接提交到 GitHub。
- 涉及真实厂区、人员、事故、设备铭牌或敏感点位的图片必须先脱敏。
- 文件命名应表达内容含义，避免 `1.png`、`new.png`、`test.jpg` 等无意义命名。

## 维护建议

新增资源时同步更新相关文档，说明该资源服务于哪个页面、报告或算法验证流程。无法明确用途的资源默认不进入仓库。
