# PINN 气体泄漏反演模型问题分析

> 当前代码并非真正训练型 PINN，而是"高斯烟羽物理模型 + 粗网格搜索 + Nelder-Mead 局部优化 + 人为打分项"的组合。

---

## 核心问题（按严重程度排序）

### 1. 这个"PINN"并没有真正训练神经网络

虽然配置里有 `epochs`、`learningRate`，但实际反演流程中没有神经网络参数训练：

- `run_pinn_inversion_task` 只做 payload normalize，然后调用 `run_two_stage_inversion`
- 真正优化位置时使用的是 `nelder_mead_2d` — 传统数值优化器，不是神经网络训练
- 它不是"训练不充分"，而是"当前根本不是训练型 PINN"

### 2. 风向大概率有问题 — **最高优先级**

代码注释写：

```
wind_direction_deg: Wind direction in degrees (FROM direction)
```

即"风从哪个方向来"。但代码直接把这个角度当成扩散传播方向计算 `along`。

如果前端传的是气象意义的"风向"（例如东风 = 风从东往西吹），那么扩散方向应该是：

```
transportDirection = (windDirection + 180) % 360
```

如果这里没做转换，模型会把上风向和下风向搞反，结果会偏得很严重。**这个比"没训练"还致命。**

### 3. 粗搜索损失函数太弱，容易选错候选区

粗搜索中每个候选点先算所有传感器预测值，然后用：

```
norm_predicted = raw_predictions[i] / max_predicted
norm_observed = observedSignal / max_observed
```

把预测和观测都归一化后再比较。问题是：

- **只比较形状，不比较真实浓度大小**
- 导致很多错误位置也能得高分
- 传感器数量少、布局不均匀、风向错误时，粗搜索一开始就会把候选区带偏

### 4. 精修阶段被粗搜索候选区锁死了

`refine_candidate` 只围绕最佳候选区域做局部优化，边界是：

```
candidate_center ± candidate_radius * 2.5
```

如果粗搜索第一名错了，精修基本只能在错误区域附近打转。它不是全局重新找，而是：

1. 先相信粗搜索
2. 再局部微调

**粗搜索错，后面就很难救回来。**

### 5. 传感器信号用了历史峰值，可能和当前风场不匹配

粗搜索里信号是：

```
observed_signal = peak * 0.65 + current * 0.35
```

反演数据里也类似：

```
signal = peak * 0.55 + current * 0.45
```

如果当前帧气体已经被风吹走了，但某个传感器历史峰值很高，模型还会认为它当前也很重要。这会造成：

- **当前风场** 和 **传感器信号** 不是同一个时刻的物理状态
- 反演自然会偏

### 6. PDE loss 也不是真正 PDE 约束

现在的 `compute_pde_loss` 本质是在惩罚"上风向传感器有信号"和"横风偏移太大"。它不是用神经网络去最小化对流扩散方程残差，更像：

- ❌ 严格 PINN 里的物理方程残差约束
- ✅ 风向规则惩罚项

---

## 结论

找不到泄漏点不是单一原因造成的，最可能是以下几个叠加：

1. 风向语义可能反了
2. 粗搜索只做归一化形状匹配，候选区容易错
3. 精修只在第一候选区附近局部搜索
4. 信号混入历史峰值，和当前风场不一致
5. 当前实现不是真正训练型 PINN

---

## 修复优先级

### 第一优先级：风向

如果 `windDirection` 表示"风从哪里来"，模型内部传播方向使用：

```
transportDirection = (windDirection + 180) % 360
```

**这个最可能立刻改善结果。**

### 第二优先级：不要只 refine 第一候选区

改成：

1. topK 候选区全部 refine
2. 最后按 `finalLoss` 排序
3. 选最优

现在只选 `ranked_candidates[0]`，太容易被粗搜索第一名误导。

### 第三优先级：信号先只用 current，不用 peak

先改成：

```
signal = current
```

不要混历史峰值。等模型稳定后，再考虑：

```
signal = 0.8 * current + 0.2 * peak
```

---

## 关于"要不要训练"

### 如果你只是做项目演示

**不用真的训练 PINN。** 先把上面三个工程问题修好，效果会明显提升。

### 如果你想做得更像论文/真正模型

需要：

1. 用仿真数据生成大量 **泄漏源-风场-传感器浓度** 样本
2. 训练一个神经网络回归 `source_x / source_y / Q`
3. 再用物理损失约束扩散方程

这是另一个大工程。当前代码不是这个路线。
