## 五大核心模块

### 1. 气体实时监控
传感器采集园区环境数据（气体浓度、温湿度），经后端清洗处理后存入时序数据库，前端实时渲染展示。

### 2. 分级及时预警
配置多级预警阈值，异常数据自动触发告警，联动溯源模块启动应急流程。

### 3. 泄漏源估计
基于多点浓度观测、物理扩散模型和 EKI/PINN 等反演方法估计候选泄漏源位置，并输出误差、置信信息和适用边界。CFD 可作为后续高保真验证方向，不在当前仓库中冒充已完成的生产级能力。

### 4. 三维可视化
优先接入 SuperMap iPortal 已建设数字大屏，展示园区三维场景、监控点位、巡检小车、泄漏点和险情态势；Three.js/SuperMap 原生三维能力作为后续扩展，Canvas 承载二维态势地图。

### 5. 智能小车调度
D\* Lite 算法实现动态应急避障路径规划，派遣小车现场采样复核。

## 算法模块

| 算法 | 模型 | 用途 |
|------|------|------|
| 气体扩散 | 经典/改进高斯烟羽与烟团模型 | 模拟 CO、O2、NH3、CH4 浓度扩散趋势 |
| 参数标定 | K 网格搜索 + FB/NMSE/FAC2 | 基于观测数据客观选择 `diffusionCorrectionK` |
| 泄漏溯源 | EKI/PINN 反演 | 估计候选泄漏源位置，输出误差和置信信息 |
| 路径规划 | D\* Lite | 根据危险浓度区域生成和更新逃生路线 |
| 点位布局 | 规则约束 + 后续优化模型 | 依据标准和风险区域优化监控点位布局 |
| 视觉识别 | YOLO11m | 识别小车摄像头图像中的人员位置 |
| 高保真校验 | CFD/实测数据对比 | 后续验证方向，需单独记录数据来源和边界 |

## 数据流转

```
传感器/小车 → Java 后端 → Python 扩散模型 → 参数标定/溯源 →
D* Lite 路径规划 → 前端 Canvas 与 SuperMap iPortal 展示 →
告警联动与应急建议 → 记录归档与后续验证
```
