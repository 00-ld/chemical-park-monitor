# 传感器风险等级划分标准

本文档约束当前系统内传感器布点风险评分的解释口径。项目当前硬件检测范围限定为四类气体：一氧化碳（CO）、甲烷（CH4）、氨气（NH3）和氧气（O2）。如后续增加其他危险气体，必须同步更新前端配置、后端种子数据、算法参数和本文档。

## 1. 风险评分原则

风险评分用于辅助传感器点位优先级排序，不等同于工程验收结论。实际布点仍需结合 GB/T 50493、GB 18218、园区工艺资料、设备台账、风场条件和现场安全评审。

核心简化公式：

```text
R = alpha * beta * (q / Q) * locationFactor
```

| 参数 | 含义 | 当前取值依据 |
| --- | --- | --- |
| alpha | 暴露人员校正系数 | 根据区域人员密度估算，范围 0.5~2.0 |
| beta | 气体危险性校正系数 | 参考 GB 18218 毒性/危险性修正思想 |
| q / Q | 存在量与临界量比值 | 由设施 hazardLevel 映射得到 |
| locationFactor | 安装位置修正 | 根据安装高度和设施邻近关系估算 |

## 2. 风险等级

| 等级 | 名称 | R 值范围 | 系统用途 |
| --- | --- | --- | --- |
| 1 | 重大风险 | R >= 50 | 必须重点布点，优先报警和联动 |
| 2 | 较大风险 | 10 <= R < 50 | 建议重点布点，纳入巡检关注 |
| 3 | 一般风险 | 5 <= R < 10 | 常规布点，保持数据留存 |
| 4 | 低风险 | R < 5 | 辅助覆盖，按需巡检 |

## 3. 当前四气 beta 映射

| 气体 | beta | 说明 |
| --- | ---: | --- |
| NH3 氨气 | 2.0 | 有毒刺激性气体，当前项目重点有毒气体 |
| CO 一氧化碳 | 2.0 | 有毒气体，需重点监测 |
| CH4 甲烷 | 1.5 | 可燃气体，按易燃气体处理 |
| O2 氧气 | 1.0 | 缺氧/富氧监测，主要影响人员安全和燃烧风险 |

当一个传感器检测范围写作 `CO/CH4/NH3/O2` 时，系统按其中较高风险气体计算 beta，当前取值为 `2.0`。

## 4. q/Q 映射

传感器本身不含危化品储量。系统以设施 `hazardLevel` 作为近似输入：

```text
q / Q = 0.5 + hazardLevel * 9.5
```

| hazardLevel | q/Q | 典型说明 |
| ---: | ---: | --- |
| 1.0 | 10.0 | 液氨储罐、核心危化品设备 |
| 0.7 | 7.15 | 高风险生产装置 |
| 0.5 | 5.25 | 中等风险辅助或存储设施 |
| 0.3 | 3.35 | 低风险公用工程 |
| 0.15 | 1.93 | 办公或低危辅助区域 |

## 5. 示例

### 5.1 液氨储罐旁传感器

```text
detectionRange = "CO/CH4/NH3/O2"
installationHeight = 2.2m
nearestFacility = T-07 氨气储罐
hazardLevel = 1.0

beta = 2.0
alpha = 1.0
q/Q = 0.5 + 1.0 * 9.5 = 10.0
locationFactor = 1.05

R = 1.0 * 2.0 * 10.0 * 1.05 = 21.0
风险等级 = 2 级较大风险
```

### 5.2 生产车间周边传感器

```text
detectionRange = "CO/CH4/NH3/O2"
installationHeight = 1.5m
nearestFacility = 高风险生产装置
hazardLevel = 0.7

beta = 2.0
alpha = 1.2
q/Q = 0.5 + 0.7 * 9.5 = 7.15
locationFactor = 1.0

R = 1.2 * 2.0 * 7.15 * 1.0 = 17.16
风险等级 = 2 级较大风险
```

### 5.3 办公区边界传感器

```text
detectionRange = "CO/CH4/NH3/O2"
installationHeight = 1.5m
nearestFacility = 办公设施
hazardLevel = 0.15

beta = 2.0
alpha = 1.5
q/Q = 0.5 + 0.15 * 9.5 = 1.925
locationFactor = 1.0

R = 1.5 * 2.0 * 1.925 * 1.0 = 5.78
风险等级 = 3 级一般风险
```

## 6. 维护要求

- 风险评分只作为安全决策辅助，严禁写成绝对可靠结论。
- 修改四气范围、beta、阈值或设施 hazardLevel 时，必须同步更新算法、前端展示、数据库种子和接口文档。
- 引入真实工程验收前，需要由具备资质的安全评估人员复核布点规则和标准引用。
