# PINN 泄漏源溯源算法改进方案

## 一、现状分析

### 当前实现

| 模块 | 文件 | 功能 |
|------|------|------|
| 粗搜索 | `python/inversion/pinn_coarse_search.py` | 固定 8×6 网格扫描候选泄漏区域 |
| PINN 模型 | `python/inversion/pinn_model.py` | 4 层 64 神经元全连接网络 |
| 数据集 | `python/inversion/pinn_dataset.py` | 从传感器读数提取训练样本 |
| 损失函数 | `python/inversion/pinn_losses.py` | 数据损失 + 物理约束损失 |
| 反演主逻辑 | `python/inversion/source_inversion.py` | 协调粗搜索 + PINN 求解 |
| 运行器 | `python/inversion/inversion_runner.py` | Pyodide Worker 入口 |

### 现存问题

1. **传感器数量不足** — 默认 10-15 个传感器，反演约束太少
2. **粗搜索网格太粗** — 步长大，候选区域定位精度 ±50m
3. **PINN 训练不足** — 500 轮迭代，损失函数未充分收敛
4. **无时序融合** — 仅用单帧浓度场，未利用扩散动画多帧信息
5. **无置信度输出** — 结果单一位置点，评委无法评估可靠性

## 二、改进方案（按优先级排序）

### 方案 1：虚拟传感器插值

**目标：** 将有效观测点从 10-15 个扩展到 50-80 个

**方法：** 在实测传感器之间进行径向基函数（RBF）插值

```
实测传感器:  S1, S2, S3, ... (10-15个)
                    ↓ RBF 插值
虚拟传感器:  V1, V2, V3, ... + 实测 (50-80个)
                    ↓
               PINN 输入维度 ↑ 5-8x
               反演约束 ↑ 定位精度 ↑
```

**涉及文件：** `python/inversion/pinn_dataset.py`
**新增函数：** `interpolate_virtual_sensors(real_sensors, grid_density=5)`
**预估效果：** 定位误差 ±50m → ±30m
**实现难度：** 低

### 方案 2：自适应网格粗搜索

**目标：** 提高候选区域定位精度

**当前：** 
- 固定 8×6 网格，每个单元格大小约 125×108
- 候选区域：取浓度残差最小的几个单元格

**改进后（两步法）：**

```
第一步：大步长全局搜索（当前方案的 8×6）
    ↓ 取 Top-3 候选区域
第二步：在候选区域局部细化（4 倍密度，即 16×12 子网格）
    ↓ 合并相邻高分区
最终候选区域：精度提升 4 倍
```

**涉及文件：** `python/inversion/pinn_coarse_search.py`
**预估效果：** 定位误差 ±30m → ±20m
**实现难度：** 中

### 方案 3：PINN 训练增强

**目标：** 提升神经网络反演精度

| 参数 | 当前值 | 改进值 |
|------|--------|--------|
| 训练轮次 | 500 | 2000+ |
| 网络层数 | 4 | 6 |
| 神经元数 | 64 | 128 |
| 激活函数 | tanh | tanh + residual |
| 学习率 | 1e-3 | 1e-3 → 1e-5 衰减 |
| 早停机制 | 无 | patience=200 |
| 物理损失权重 | 固定 0.1 | 自适应 0.1→0.5 |

**涉及文件：**
- `python/inversion/pinn_model.py` — 网络结构改进
- `python/inversion/pinn_losses.py` — 损失权重自适应
- `python/inversion/source_inversion.py` — 训练循环

**预估效果：** 定位误差 ±20m → ±15m
**实现难度：** 中

### 方案 4：多帧时序融合

**目标：** 利用扩散动画的帧序列信息，而非单帧

**方法：**

```
当前：  frame[t] → PINN → 单一位置估计

改进：
  frame[t-2], frame[t-1], frame[t], frame[t+1], frame[t+2] 
                        ↓ 时序编码器
              融合特征向量
                        ↓ PINN
              位置估计 + 置信区间
```

**前端改动：**
- `algorithm.ts` 新增 `runPinnInversionSequence()` API
- `map_test.vue` 新增"时序反演"按钮

**后端改动：**
- `python/inversion/source_inversion.py` 新增时序处理逻辑
- `python/inversion/pinn_model.py` 新增时序编码器

**预估效果：** 定位误差 ±15m → ±10m
**实现难度：** 较高

### 方案 5：贝叶斯不确定性量化

**目标：** 输出置信椭圆而非单一位置点，让评委看到"可信度"

**方法：**

```
训练完成后，开启 MC Dropout：
  for i in 1..20:
      output_i = model(input, dropout=True)  随机丢弃神经元
  
  最终输出:
    mean_x = mean(output_i.x)           # 平均位置
    mean_y = mean(output_i.y)
    sigma_x = std(output_i.x)           # 标准差 → 椭圆半轴
    sigma_y = std(output_i.y)
    confidence = 1 - max(sigma_x, sigma_y) / threshold  # 置信度 0~1
```

**前端改动：**
- 在 Canvas 上绘制置信椭圆（已有 `drawPinnConfidenceEllipse` 函数框架）

**后端改动：**
- `python/inversion/pinn_model.py` 新增 MC Dropout 推理模式

**预估效果：** 输出包含统计意义的置信区间
**实现难度：** 高

### 方案 6：风场自适应校准

**目标：** 利用传感器读数反推局部风场偏差

**方法：**
1. 比较传感器实测浓度与高斯烟羽模型预测浓度
2. 计算局部风场修正量（风向偏差 ±5-15°，风速系数 0.8-1.2）
3. 将修正后风场输入 PINN

**涉及文件：** `python/inversion/source_inversion.py`

**预估效果：** 在复杂地形场景下定位误差额外降低 10-20%
**实现难度：** 高

## 三、实施路线图

```
Phase 1（1-2 天）     Phase 2（2-3 天）       Phase 3（3-5 天）
─────────────────    ─────────────────       ─────────────────
虚拟传感器插值        多帧时序融合              贝叶斯不确定性量化
自适应粗搜索          PINN 训练增强             风场自适应校准
                     └── 前后端联调
```

### 推荐优先实施顺序

1. **虚拟传感器插值** — 性价比最高，改1个文件，误差降 40%
2. **自适应粗搜索** — 改1个文件，误差进一步降 33%
3. **PINN 训练增强** — 改3个文件，误差再降 25%
4. **时序融合 + 贝叶斯** — 较大改动，效果显著但实现周期长
5. **风场校准** — 锦上添花，按需实施
