# 系统架构文档


## 五大核心模块

### 1. 氛围实时监控
传感器采集园区环境数据（气体浓度、温湿度），经后端清洗处理后存入时序数据库，前端实时渲染展示。

### 2. 分级及时预警
配置多级预警阈值，异常数据自动触发告警，联动溯源模块启动应急流程。

### 3. 泄漏精准溯源
PINN 物理神经网络 + CFD 流体力学双模型验证，精准定位泄漏源坐标与扩散范围。

### 4. 三维可视化
基于 Three.js 渲染园区三维场景、气体云图、险情热力图，Canvas 绘制二维态势地图。

### 5. 智能小车调度
D\* Lite 算法实现动态应急避障路径规划，派遣小车现场采样复核。

## 算法模块

| 算法 | 模型 | 用途 |
|------|------|------|
| 气体扩散 | 改进高斯烟羽模型 | 模拟有害气体浓度扩散趋势 |
| 泄漏溯源 | PINN 物理神经网络 | 精准定位泄漏源头 |
| 仿真校验 | CFD 流体力学模型 | 验证溯源结果精准度 |
| 路径规划 | D\* Lite | 智能小车动态应急避障调度 |
| 点位优化 | RL 强化学习 | 优化园区监控点位布局 |
| 视觉识别 | YOLO11 | 识别明火、设备异常、违规隐患 |

## 数据流转

```
传感器 → Java 后端 → Python 扩散模型 → PINN 溯源 →
CFD 校验 → 前端 Canvas/Three.js 渲染 → 小车调度系统 →
YOLO 视觉识别 → RL 点位优化 → 闭环反馈
```
